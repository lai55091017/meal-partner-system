# 飯搭子系統：按鈕轉跳與功能說明

## 開啟方式
先cd "D:\Owner\桌面\meal-partner-system\meal-partner-backend"
↓
後端啟動 npm run dev
↓
server.js 只是開始監聽
↓
不會自動執行 schema.sql
↓
必須有人打開 http://localhost:3000/api/init-db
↓
才會建立資料表
## 1. 頁面結構

本專案是單頁式前端 Prototype，所有頁面都寫在 `index.html` 裡面，再由 `app.js` 控制顯示哪一個頁面。

| 頁面名稱 | HTML id | 功能 |
|---|---|---|
| 首頁 | `view-home` | 顯示我的飯局、其他飯局、搜尋列、餐期篩選 |
| 通知 | `view-notifications` | 顯示系統通知 |
| 聊天室 | `view-chat` | 顯示聊天室列表 |
| 飯局詳情 | `view-party-detail` | 顯示被點選的飯局資料，並可按 join |
| 已加入飯局 | `view-party-joined` | 顯示飯局資料與已加入成員 |
| 評價 | `view-rating` | 對加入成員進行星等與文字評價 |
| 新增飯局 | `view-create` | 輸入飯局名稱、約飯人、店家、時間 |
| 個人帳號 | `view-profile` | 顯示個人資料、評價、登出按鈕 |

## 2. 按鈕與轉跳流程

| 使用者操作 | 觸發元素 | 轉跳頁面 | JS 函式 |
|---|---|---|---|
| 點底部「通知」 | `.nav-item[data-nav="notifications"]` | 通知頁 | `switchView("notifications")` |
| 點底部「聊天室」 | `.nav-item[data-nav="chat"]` | 聊天室 | `switchView("chat")` |
| 點底部「home」 | `.nav-item[data-nav="home"]` | 首頁 | `switchView("home")` |
| 點底部「個人帳號」 | `.nav-item[data-nav="profile"]` | 個人帳號 | `switchView("profile")` |
| 點首頁飯局卡片 | `#my-party-card` / `#other-party-card` | 飯局詳情 | `openPartyDetail()` |
| 點詳情頁 join | `#party-join-btn` | 已加入飯局 | `openJoinedParty()` |
| 點已加入頁 評價 | `#party-rate-btn` | 評價頁 | `switchView("rating")` |
| 點右下角 + | `#fab` | 新增飯局 | `switchView("create")` |
| 新增飯局送出 | `#create-form` | 首頁 | `getPartyDataFromForm()` + `addMyParty()` + `renderMyParties()` |
| 點篩選按鈕 | `#filter-btn` | 展開 / 收合篩選 | `initFilterMenu()` |
| 點星星 | `.star` | 不換頁，只改評分 | `setStarRating()` |
| 點 sign out | `#profile-signout` | 不換頁，跳出確認 | `confirm()` + `alert()` |

## 3. 重要 JS 函式說明

### `switchView(viewKey)`
控制所有頁面的顯示與隱藏。它會把目標頁面顯示出來，其他頁面全部隱藏。

### `setActiveNav(activeNavKey)`
控制底部導覽列哪一個項目要亮起。像「飯局詳情、已加入、評價、新增飯局」都屬於首頁延伸流程，所以會讓 home 維持亮起。

### `getPartyDataFromCard(card)`
從首頁飯局卡片的 `data-*` 屬性讀取飯局資料，例如飯局名稱、約飯人、店家、時間。

### `getPartyDataFromForm()`
從新增飯局表單讀取使用者輸入的資料。如果欄位空白，就使用預設文字。

### `fillPartyDetail(prefix, party)`
把飯局資料填入詳情頁或已加入頁。`prefix` 可以是 `detail` 或 `joined`。

### `openPartyDetail(card, options)`
使用者點首頁卡片時呼叫，負責讀取飯局資料並打開飯局詳情頁。

### `openJoinedParty()`
使用者按 join 或新增飯局送出後呼叫，負責打開已加入飯局頁。

### `initFilterMenu()`
初始化首頁餐期篩選選單，支援滑鼠點擊與鍵盤 Enter / Space 操作。

### `setStarRating(group, value)`
更新星星評分，會改變星星樣式與 `data-rating` 數值。

### 新增飯局表單擴充

1. 人數上限。

使用者點擊首頁「其他飯局」後，進入飯局詳情頁，按下 `join` 會：

5. 將加入後的飯局資料存到 `localStorage`。
6. 自動切換到「已加入飯局 / 成員頁」。

## 8. 本次新增：退出飯局功能

本版本在「已加入飯局 / 成員頁」新增 `退出飯局` 按鈕，讓已加入其他飯局的使用者可以退出。

### 功能流程

| 使用者操作 | 系統反應 |
|---|---|
| 使用者加入其他飯局 | 自己會被加入成員列表，飯局人數 +1 |
| 進入已加入飯局頁 | 會看到聊天室、退出飯局、評價按鈕 |
| 點擊「退出飯局」 | 系統會詢問是否確定退出 |
| 確認退出 | 從成員列表移除目前使用者 |
| 退出後 | 飯局目前人數自動 -1 |
| 退出後 | 回到飯局詳情頁，`join` 按鈕恢復可點擊 |
| 退出後再次按 join | 可以重新加入該飯局 |

### 程式重點

| 函式 | 功能 |
|---|---|
| `canCurrentUserLeaveParty(party)` | 判斷目前登入者是否可以退出飯局，主辦人不可退出 |
| `leaveCurrentParty()` | 移除目前使用者、更新人數、更新 localStorage |
| `updateJoinedActionButtons(party)` | 控制「退出飯局」按鈕是否顯示 |
| `renderJoinedMembers(party)` | 重新渲染成員列表，退出後不再顯示自己 |

### 資料儲存

退出飯局後，系統會更新 `localStorage` 中的 `mealPartnerJoinedParties`，因此重新整理頁面後，成員列表與人數仍會保持退出後的狀態。

### 展示說明

可以在報告中這樣說：

> 本次新增退出飯局功能，使用者在加入飯局後，可以於成員頁點擊「退出飯局」。系統會自動將目前使用者從成員列表移除，並同步更新飯局人數。退出後會回到飯局詳情頁，join 按鈕會恢復，使用者仍可再次加入該飯局。此功能讓飯局流程更加完整，包含加入、查看成員、聊天與退出。


## 本次修正：首頁其他飯局顯示人數

修正「其他飯局」卡片未顯示目前人數的問題。

- 其他飯局卡片現在會顯示 `目前人數 / 人數上限`，例如 `1 / 4 人`。

## 7. 本次新增：搜尋與篩選飯局

本版本新增首頁搜尋與篩選功能，讓首頁飯局卡片可以依條件即時更新。

| 功能 | 說明 |
|---|---|
| 關鍵字搜尋 | 可搜尋飯局名稱、主辦人、店家名稱 |
| 餐期篩選 | 可選全部、早餐、午餐、晚餐、宵夜 |
| 人數篩選 | 勾選「只顯示尚未額滿」後，只會顯示目前人數小於人數上限的飯局 |
| 即時更新 | 搜尋框輸入文字、切換餐期、勾選未額滿時，首頁會立即重新顯示符合條件的飯局 |
| 無結果提示 | 如果沒有符合條件的飯局，會顯示「目前沒有符合條件的飯局」 |

### 主要新增 DOM

| 元素 | 功能 |

| `#available-only-filter` | 只顯示尚未額滿飯局 |
| `#home-no-result` | 無搜尋結果提示 |

### 主要新增 JS 函式

| 函式 | 功能 |
|---|---|
| `inferMealType(party)` | 判斷飯局餐期，優先使用資料中的 `mealType`，沒有時會從飯局名稱、時間、介紹推測 |

| `matchesPartyFilters(party)` | 判斷單一飯局是否符合目前篩選條件 |
| `renderHomeParties()` | 重新渲染首頁我的飯局與其他飯局 |
| `updateHomeNoResult()` | 沒有任何符合條件的飯局時顯示提示文字 |

### 建立飯局新增欄位

新增飯局表單增加 `餐期` 欄位，使用者建立飯局時可以選擇早餐、午餐、晚餐或宵夜，之後首頁篩選會依照這個欄位過濾飯局。

## 7. 本次新增：飯局狀態顯示

本版本新增飯局狀態判斷，首頁卡片與飯局詳情頁都會顯示狀態標籤。

| 狀態 | 顯示條件 | 使用者能否加入 |
|---|---|---|
| 招募中 | 尚未額滿，且目前使用者尚未加入 | 可以加入 |
| 已加入 | 目前使用者已經加入該飯局 | 不重複加入，可查看成員 |
| 已額滿 | 成員人數已達人數上限 | 不能加入 |
| 我是主辦人 | 該飯局是目前使用者建立的飯局 | 不需要 join |

### 程式重點

- `getPartyStatus(party)`：依照目前使用者、成員列表與人數上限判斷狀態。
- `setStatusClass(element, status)`：更新狀態文字與 CSS 樣式。
- `createPartyCard()`：建立首頁我的飯局卡片時加入狀態標籤。
- `prepareOtherPartyCard()`：更新其他飯局卡片時同步顯示狀態。
- `openPartyDetail()`：進入詳情頁時依狀態調整 `join` 按鈕，例如已額滿會停用，主辦人會顯示「我是主辦人」。

## 8. 本次新增：取消飯局功能

本版本新增主辦人取消飯局功能：

| 功能 | 說明 |
|---|---|
| 取消飯局 | 主辦人可在飯局成員頁點擊「取消飯局」 |
| 狀態更新 | 取消後飯局狀態會變成「已取消」 |
| 停用加入 | 已取消飯局的 `join` 按鈕會停用，其他人不能再加入 |
| 保留資料 | 飯局卡片、詳情、人數與成員列表仍會保留，方便查看紀錄 |
| 儲存狀態 | 取消狀態會儲存在 `localStorage`，重新整理後仍會顯示已取消 |

取消後，首頁飯局卡片與飯局詳情頁都會顯示「已取消」狀態；若勾選「只顯示尚未額滿」，已取消飯局不會被視為可加入飯局。
