# 飯搭子系統測試流程

這份測試流程用來確認系統主要功能是否都已成功連接 PostgreSQL。

## 一、測試前準備

1. 啟動 PostgreSQL。
2. 啟動後端：

```powershell
cd "D:\Owner\桌面\meal-partner-system\meal-partner-backend"
npm run dev
```

3. 確認後端正常：

```text
http://localhost:3000/api/db-test
```

4. 若尚未建立資料表，執行：

```text
http://localhost:3000/api/init-db
```

5. 開啟前端 Live Server。

---

## 二、註冊 / 登入測試

### 測試步驟

1. 開啟前端。
2. 點擊「個人帳號」。
3. 切換到「註冊」。
4. 輸入：

```text
帳號：test001
姓名：王小明
密碼：1234
確認密碼：1234
```

5. 按下 Register。
6. 登入後確認可進入個人帳號頁。

### 資料庫確認

```sql
SELECT id, account, name, student_id, department, created_at
FROM users
ORDER BY id;
```

---

## 三、個人資料與飲食偏好測試

### 測試步驟

1. 進入個人帳號頁。
2. 點擊「編輯」。
3. 修改姓名、系所、個人介紹。
4. 勾選飲食偏好。
5. 按「儲存」。
6. 重新整理頁面後確認資料仍存在。

### 資料庫確認

```sql
SELECT id, account, name, student_id, department, bio
FROM users
ORDER BY id;

SELECT *
FROM user_preferences
ORDER BY user_id, type, value;
```

---

## 四、建立飯局測試

### 測試步驟

1. 使用帳號 A 登入。
2. 點擊右下角「+」。
3. 建立一筆飯局。
4. 回到首頁，確認「我的飯局」出現新飯局。

### 資料庫確認

```sql
SELECT *
FROM parties
ORDER BY id;

SELECT *
FROM party_members
ORDER BY party_id, user_id;
```

---

## 五、加入飯局測試

### 測試步驟

1. 使用帳號 B 登入。
2. 在「其他飯局」找到帳號 A 建立的飯局。
3. 點擊飯局並按 join。
4. 確認人數增加。

### 資料庫確認

```sql
SELECT *
FROM party_members
ORDER BY party_id, user_id;
```

---

## 六、聊天室測試

### 測試步驟

1. 進入已加入飯局。
2. 點擊聊天室。
3. 傳送一則訊息。
4. 重新整理頁面。
5. 再次進入聊天室，確認訊息仍存在。

### 資料庫確認

```sql
SELECT *
FROM chat_messages
ORDER BY id;
```

---

## 七、通知測試

### 測試步驟

1. 建立飯局。
2. 加入飯局。
3. 退出飯局。
4. 傳送聊天室訊息。
5. 進入通知頁，確認通知出現。
6. 重新整理頁面後確認通知仍存在。
7. 刪除通知。

### 資料庫確認

```sql
SELECT *
FROM notifications
ORDER BY id DESC;
```

---

## 八、取消與刪除飯局測試

### 測試步驟

1. 使用主辦人帳號進入自己建立的飯局。
2. 按「取消飯局」。
3. 確認飯局狀態變成已取消。
4. 按「刪除紀錄」。
5. 回到首頁確認飯局消失。

### 資料庫確認

```sql
SELECT *
FROM parties
ORDER BY id;
```

若刪除成功，該筆飯局不應該出現在 `parties` 表。

---

## 九、評價測試

### 測試步驟

1. 使用帳號 A 建立飯局。
2. 使用帳號 B 加入飯局。
3. 飯局時間結束後進入飯局。
4. 點擊「評價」。
5. 給成員星等與留言。
6. 送出評價。
7. 進入個人帳號頁確認平均評分與評價紀錄。
8. 回到同一場飯局，確認不能重複評價。

### 資料庫確認

```sql
SELECT *
FROM ratings
ORDER BY id;
```

---

## 十、清空測試資料

若要清空飯局、通知、聊天與評價資料，但保留使用者帳號：

```sql
TRUNCATE TABLE chat_messages, ratings, notifications, party_members, parties
RESTART IDENTITY CASCADE;
```

若也要清空使用者與偏好：

```sql
TRUNCATE TABLE chat_messages, ratings, notifications, party_members, parties, user_preferences, users
RESTART IDENTITY CASCADE;
```
